
# COMrade Visual Design Language
### (Logo + UI Style Guide)

## Overview

COMrade is a **USB HID / Serial data monitor and logger** for macOS.  
The visual identity combines three themes:

1. **Low-level developer tooling** (terminal prompts, data streams)
2. **Hardware communication** (USB symbol, signal monitoring)
3. **Bold industrial / constructivist aesthetics** (diagonal shapes, strong contrast)

The result should feel like:

- A **powerful engineering utility**
- Built by **systems programmers**
- Slightly **retro-technical**, but modern and polished

The UI should feel somewhere between:

- **macOS developer tools** (Console, Instruments, Activity Monitor)
- **firmware debugging software**
- **hardware engineering instrumentation**

---

# Core Icon Structure

The COMrade icon is a **rounded-square macOS app icon** with strong geometric elements.

### Layout

The icon contains four primary visual elements:

1. **Diagonal Red Field**
2. **Dark System Background**
3. **USB Trident Symbol**
4. **Terminal Prompt (`>_`)**
5. **COMRADE wordmark**

The icon is intentionally **bold and graphic**, with minimal small detail.

---

# Symbol Meaning

### USB Trident

Represents:

- USB communication
- hardware interface
- device connectivity

It is the **central symbol of the app**.

It should always appear **centered and prominent** in branding.

---

### Terminal Prompt

```
>_
```

Represents:

- developer tooling
- command line workflows
- data inspection

This reinforces that COMrade is **a serious engineering utility**, not consumer software.

---

### Star Accent

The small star in the icon references **constructivist visual design** (industrial poster style).  
It should be used sparingly — mostly for branding or decorative accents.

---

# Color Language

The icon uses a **high-contrast engineering palette**.

### Primary Colors

Deep Charcoal  
`#14171A`

Signal Red  
`#D72626`

Orange Gradient Accent  
`#FF6A00`

Cream / Ivory  
`#F1E6CF`

---

### Color Roles

| Color | Meaning |
|------|--------|
| Charcoal | system background |
| Red | power, signal, monitoring |
| Orange | activity / motion |
| Cream | hardware / symbols |

---

### UI Usage

Charcoal should be the **primary UI background**.

Red/orange should be used for:

- highlights
- active signals
- graphs
- event indicators
- device activity

Cream should be used for:

- iconography
- labels
- hardware symbols

---

# Shape Language

The icon relies heavily on **diagonal geometry**.

### Key design motif

```
╱
```

Diagonal divisions represent:

- signal flow
- data movement
- electrical pathways

These diagonals should appear in:

- section dividers
- page backgrounds
- accent panels
- hero areas on the landing page

---

# Typography

Typography should feel **technical and strong**.

### Headings

Bold geometric sans-serif

Good candidates:

- Inter
- IBM Plex Sans
- SF Pro Display
- Space Grotesk

Example style:

```
COMRADE
```

or

```
COMrade
```

Where **COM** is emphasized.

---

### Body text

Clean modern sans-serif:

- Inter
- SF Pro
- IBM Plex Sans

---

### Code / Data

Use monospaced fonts:

- JetBrains Mono
- IBM Plex Mono
- SF Mono

---

# Texture / Background

Background surfaces should be **dark and slightly industrial**.

Acceptable treatments:

- subtle brushed metal
- matte dark panels
- faint grid overlays
- data waveform patterns

Avoid:

- bright gradients
- playful visuals
- soft pastel colors

---

# UI Mood

The software should feel like:

**A hardware engineer's diagnostic instrument**

Keywords:

- precise
- powerful
- confident
- industrial
- technical
- focused

Avoid:

- playful UI
- consumer app aesthetics
- excessive animations

---

# Landing Page Direction

The landing page should visually echo the icon.

### Hero Section

Large diagonal split background.

Example structure:

```
╱ RED SIGNAL FIELD
╲ DARK SYSTEM FIELD
```

Center:

```
USB symbol
COMrade
USB HID + Serial Monitor
```

Below that:

Live style visuals such as:

- waveform data
- device activity logs
- serial output streams

---

# Motion Design

Animations should feel like **signal activity**.

Examples:

- scrolling serial logs
- blinking cursor
- waveform pulses
- USB device connect indicators

Avoid:

- playful micro-interactions
- excessive UI transitions

---

# UI Components

### Buttons

Rectangular with subtle corner rounding.

Active state:

Red or orange glow.

---

### Panels

Dark cards with thin borders.

`#24292E`

---

### Data Views

Terminal-style.

Example:

```
[12:04:12] USB HID Device Connected
[12:04:12] VendorID: 0x1234
[12:04:13] Packet Received
```
